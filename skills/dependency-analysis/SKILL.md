# 依存関係分析

内部モジュール間および外部ライブラリの依存関係を分析・可視化する。

## 制約（カテゴリ共通）

- コードを一切変更しない（read-only）
- 成果物は必ず分析ドキュメントとして残す（頭の中の理解で終わらせない）

## 前提入力

**以下の必須入力が揃わない場合、作業を開始せず、不足しているものを具体的に挙げてユーザーに要求すること。推測で補完しない。**

- 【必須】分析のスコープ（内部依存か外部依存か両方か、対象範囲）
- 【必須】依存関係の情報源（パッケージ定義ファイル、import文等の所在）
- 【任意（あれば使う）】依存方向のルール（層は下向きのみ等）。プロジェクトのCLAUDE.md・Context文書から読み取る。見つからなければルール照合はスキップし、その旨を成果物に明記する

## 手順

1. スコープ内の依存を機械的に列挙する（宣言された依存＋実際の参照）
2. 依存を分類する（本体/開発用、直接/推移的、安定/変動の激しいもの）
3. 依存グラフを整理し、注目点（循環依存・集中点・孤立点）を抽出する
4. 注入されたルールがあれば、それに対する違反箇所を事実として列挙する
5. 分析ドキュメントにまとめる

## 成果物

依存関係分析ドキュメント（依存一覧＋グラフ＋注目点）を必ずファイルとして保存する。会話内の回答だけで終わらせない。保存先はプロジェクトの規約（CLAUDE.md等）に定めがあればそこへ。なければ保存先を提案してユーザーに確認する。

## 完了条件

スコープ内の依存が漏れなく列挙され、循環・集中などの構造的な注目点が特定されている。

## しないこと

- 依存の解消・整理の実行（→ refactor-restructure / code-cleanup を使う）
- 良し悪しの評価・改善提案（違反の「事実の列挙」までが責務。評価は checklist-code-review を使う）
