# 状態チェックポイント保存

作業の途中経過を、再開可能な形で状態ファイルに保存する。

## 制約（カテゴリ共通）

- **自由な要約を禁止する**。定型フォーマットの項目を機械的に充填するのみ（要約の質のブレが引継ぎ劣化の直接原因だからである）
- フォーマットに埋められない項目は「該当なし」「不明」と明記する（項目を黙って省略しない）

## 前提入力

**以下の必須入力が揃わない場合、作業を開始せず、不足しているものを具体的に挙げてユーザーに要求すること。推測で補完しない。**

- 【必須】状態ファイルのフォーマットと置き場所。プロジェクトのCLAUDE.md・Context文書・規約から読み取る。見つからなければ本Skill末尾のフォールバックテンプレートを使用し、置き場所はユーザーに確認する
- 【必須】現在の作業の中身（何をしていて、どこまで終わり、何が残っているか）

## 手順

1. 完了した作業を、対応する成果物ファイルのパスとともに列挙する
2. 進行中の作業について「どこまでやったか」「次の一手は何か」を具体的に記録する
3. 未着手の残作業を列挙する
4. 現時点の暫定判断・仮置きの前提（あとで見直すべきもの）を明示する
5. 状態ファイルに定型フォーマットで保存する

## 成果物

状態ファイル（更新）。必ずファイルとして保存する。保存先はプロジェクトの規約（状態管理ファイル）に定めがあればそこへ。なければ保存先を提案してユーザーに確認する。

## 完了条件

このファイルだけを読んで「次の一手」が特定できる状態になっている。

## しないこと

- 作業の続行（保存したら一旦区切る）
- 引継ぎ文書の作成（context-handover を使う。checkpointは自分向け、handoverは他者・他セッション向け）

## フォールバックテンプレート（プロジェクトに規約が無い場合のみ使用）

```markdown
# 状態チェックポイント: <日付時刻>

## 1. 完了した作業（成果物パス付き）
- <作業内容 → 成果物ファイルのパス。無ければ「該当なし」>

## 2. 進行中の作業（どこまで・次の一手）
- どこまで: <具体的な到達点。無ければ「該当なし」>
- 次の一手: <再開時に最初に行う具体的な操作。不明なら「不明」>

## 3. 未着手の残作業
- <残作業の一覧。無ければ「該当なし」>

## 4. 暫定判断・仮置きの前提
- <あとで見直すべき仮定・暫定決定。無ければ「該当なし」>
```
