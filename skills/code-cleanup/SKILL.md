# コードクリーンアップ

デッドコード・重複・未使用依存を安全に除去する。

## 制約（カテゴリ共通）

- **挙動保存**: 外から見える振る舞いを変えない。テストで確認する（テストがない場合、先にテストを整えることを要求する）
- **小さいステップで漸進**: 一度に1種類の変換のみ
- **各ステップ後に検証**: 変換→検証→次の変換、のサイクルを崩さない

## 前提入力

**以下の必須入力が揃わない場合、作業を開始せず、不足しているものを具体的に挙げてユーザーに要求すること。推測で補完しない。**

- 【必須】対象範囲と除去対象の種類（デッドコード/重複/未使用依存のどれか）
- 【必須】「使われていない」ことを確認する手段（参照検索・カバレッジ・依存解析等）
- 【必須】挙動保存を確認する手段（テストコマンド等）
- プロジェクト固有の除去判断の規約があれば、プロジェクトのCLAUDE.md・Context文書から読み取る

## 手順

1. 除去候補を機械的に列挙する（推測ではなく検索・解析に基づく）
2. 各候補について「本当に未使用か」の反証を探す（動的参照・外部からの呼び出し・設定経由の利用）
3. 確認できたものだけを1候補ずつ除去し、確認できないものは「保留リスト」に残す
4. 各除去後に検証を実行する
5. 除去一覧と保留リストを記録する

## 成果物

クリーンアップ済みコード＋除去一覧＋保留リスト。除去一覧・保留リストは必ずファイルとして保存する（会話内の回答だけで終わらせない）。保存先はプロジェクトの規約（CLAUDE.md等）に定めがあればそこへ。なければ保存先を提案してユーザーに確認する。

## 完了条件

除去したものすべてに「未使用の根拠」が記録され、検証が通っている。

## しないこと

- 「ついで」のリファクタリング（→ refactor-rename / refactor-extract / refactor-restructure を使う）
- 確証のない除去（疑わしきは保留リストへ）
