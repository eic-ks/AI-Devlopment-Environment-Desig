# リファクタリング: 抽出

大きな関数・クラス・モジュールから、凝集した単位を抽出して分割する。

## 制約（カテゴリ共通）

- **挙動保存**: 外から見える振る舞いを変えない。テストで確認する（テストがない場合、先にテストを整えることを要求する）
- **小さいステップで漸進**: 一度に1種類の変換のみ
- **各ステップ後に検証**: 変換→検証→次の変換、のサイクルを崩さない

## 前提入力

**以下の必須入力が揃わない場合、作業を開始せず、不足しているものを具体的に挙げてユーザーに要求すること。推測で補完しない。**

- 【必須】対象（分割したい関数・クラス・モジュール）と分割の動機（何が問題か）
- 【必須】モジュール分割の方針・配置規約。プロジェクトのCLAUDE.md・Context文書から読み取る。見つからなければユーザーに確認する
- 【必須】挙動保存を確認する手段（テストコマンド等）

## 手順

1. 対象の中の「一緒に変わるもの・同じ理由で存在するもの」の塊を特定する
2. 抽出単位ごとに名前・入出力（引数・戻り値・依存）を設計する
3. 1単位ずつ抽出し、元の場所からは新しい単位を呼ぶ形に置き換える
4. 各抽出後に検証を実行する
5. 抽出の結果（何をどこに抽出したか＋残った課題）を記録する

## 成果物

抽出・分割済みコード＋抽出記録。抽出記録は必ずファイルとして保存する（会話内の回答だけで終わらせない）。保存先はプロジェクトの規約（CLAUDE.md等）に定めがあればそこへ。なければ保存先を提案してユーザーに確認する。

## 完了条件

各抽出後の検証がすべて通り、元の呼び出し側の振る舞いが変わっていない。

## しないこと

- 抽出と同時の責務再配置（層をまたぐ移動は → refactor-restructure を使う）
- 抽出単位の内部ロジックの改変
