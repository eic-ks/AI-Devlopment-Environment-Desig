# 引継ぎドキュメント作成

セッション間・エージェント間の引継ぎドキュメントを、定型テンプレートの充填で作成する。

## 制約（カテゴリ共通）

- **自由な要約を禁止する**。定型フォーマットの項目を機械的に充填するのみ（要約の質のブレが引継ぎ劣化の直接原因だからである）
- フォーマットに埋められない項目は「該当なし」「不明」と明記する（項目を黙って省略しない）

## 前提入力

**以下の必須入力が揃わない場合、作業を開始せず、不足しているものを具体的に挙げてユーザーに要求すること。推測で補完しない。**

- 【必須】引継ぎテンプレート（確定事項・保留事項・次にやること・読むべきファイル等の定型項目）。プロジェクトのCLAUDE.md・Context文書・規約から読み取る。見つからなければ本Skill末尾のフォールバックテンプレートを使用する
- 【必須】現セッションの作業内容（成果物ファイル。状態ファイル＝state-checkpoint の出力があればそれを一次情報とする）
- 【任意（あれば使う）】引継ぎ先の前提（次セッションの自分か、別エージェントか）

## 手順

1. テンプレートの全項目を確認する
2. 各項目を、成果物ファイル・状態ファイルを一次情報として充填する（記憶や印象からの自由記述をしない）
3. 埋められない項目は「該当なし/不明」と明記する
4. 「次の読者が最初に読むべきファイル」のパス一覧を必ず含める
5. テンプレート全項目が埋まったことを確認して保存する

## 成果物

引継ぎドキュメント。必ずファイルとして保存する。保存先はプロジェクトの規約（引継ぎ文書の置き場）に定めがあればそこへ。なければ保存先を提案してユーザーに確認する。

## 完了条件

テンプレート全項目が充填済み（「該当なし/不明」含む）であり、自由形式の要約セクションが存在しない。

## しないこと

- 内容の要約・意訳・取捨選択の創意工夫（テンプレートが求める以上のことをしない）
- 状態ファイル自体の保存（途中経過の保存は state-checkpoint を使う）
- 一般ドキュメントの更新（documentation-update を使う）

## フォールバックテンプレート（プロジェクトに規約が無い場合のみ使用）

```markdown
# 引継ぎ: <日付・セッション識別>

## 1. 完了したこと
- <完了した作業と成果物パス。無ければ「該当なし」>

## 2. 確定した決定事項（根拠・記録先付き）
- <決定内容／根拠／記録先ファイル。無ければ「該当なし」>

## 3. 保留・要確認事項
- <未決事項・要確認事項と、確認先。無ければ「該当なし」>

## 4. 次に読むべきファイル
- <パスの一覧（読む順に）。無ければ「該当なし」>

## 5. 次にやる具体アクション
- <着手可能な粒度の具体アクション。無ければ「該当なし」>
```
