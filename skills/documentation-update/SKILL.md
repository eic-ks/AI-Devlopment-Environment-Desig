# ドキュメント更新

既存ドキュメントを、発生した変更に合わせて更新する。

## 制約（カテゴリ共通）

- **差分駆動**: 「何が変わったから、何を直す/書くか」から始める。ゼロから書き直さない（既存記録との重複・矛盾を作らないため）

## 前提入力

**以下の必須入力が揃わない場合、作業を開始せず、不足しているものを具体的に挙げてユーザーに要求すること。推測で補完しない。**

- 【必須】変更内容（diff・実装計画の完了報告・影響範囲ドキュメント等）
- 【必須】更新対象ドキュメントの一覧（またはドキュメント構成のContext）
- 【任意（あれば使う）】ドキュメントの文体・構成規約。プロジェクトのCLAUDE.md・Context文書から読み取る。見つからなければ既存ドキュメントの体裁に合わせ、判断に迷う場合はユーザーに確認する

## 手順

1. 変更内容から「ドキュメントに影響する変更点」を抽出する
2. 各変更点について、影響を受けるドキュメント箇所を特定する（全文読み直しではなく差分から逆引きする）
3. 該当箇所を更新する（変更に関係ない箇所は触らない）
4. 更新後、ドキュメント内・ドキュメント間の矛盾（古い記述の取り残し）を確認する
5. 更新一覧（どのドキュメントのどこを・なぜ）を記録する

## 成果物

更新されたドキュメント＋更新一覧。更新一覧は必ずファイルとして保存する。保存先はプロジェクトの規約に定めがあればそこへ。なければ保存先を提案してユーザーに確認する。

## 完了条件

変更点すべてにドキュメント上の対応（更新した/更新不要と判断した）が付いている。

## しないこと

- ドキュメントの新規全面書き下ろし（それは別作業として計画する）
- 引継ぎ文書の作成（引継ぎは context-handover を使う）
- 決定の記録（ADR化は decision-recording を使う）
