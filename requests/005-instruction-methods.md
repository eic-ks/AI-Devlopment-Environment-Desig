# Request 005: AIへの指示方法

- ステータス: 完了（2026-07-05）

- 依存: Phase 1（Skills）, Phase 2（Workflow）

## 目的

プロンプト単体の設計ではなく、AIとの共同開発全体における指示の出し方・タスク分割・会話の進め方・コンテキストの渡し方・レビュー依頼方法を設計する。

## 検討すべき問い

- タスクをAIに渡す時、どの粒度まで分割してから渡すべきか（大きすぎると迷子になり、小さすぎると往復コストが増える）
- 会話が長くなりそうな時、事前にどう区切っておくべきか（Phase分割の考え方をこのリポジトリの外の日常タスクにも適用できるか）
- コンテキストを渡す時、「貼り付け」「ファイル参照」「要約」のどれを使い分けるべきか
- コードレビューをAIに依頼する時、どういう依頼の仕方だと精度・再現性が上がるか（課題3「コード品質を評価できない」への対処）
- このリポジトリの`docs/handoff.md`のような構造化引継ぎは、日常のコーディングタスクにも適用できるか

## 成果物

1. 指示の出し方のベストプラクティス集
2. タスク分割の判断基準
3. コンテキストの渡し方の使い分け表
4. レビュー依頼のテンプレート・チェックリスト
