# AI開発環境・ワークフロー設計支援

## 目的

私は、AI（Claude Code、ChatGPT、Codex等）を用いたソフトウェア開発・知識管理環境を長期的に改善したい。

単なるプロンプト改善ではなく、

- AIを利用した開発ワークフロー
- Skillsの設計
- Context/RAGアーキテクチャ
- Knowledge Management
- AIへの指示方法
- コンテキスト管理
- 長期運用

まで含めた全体設計について、ベストプラクティスを提案してほしい。

目的は、

- 開発効率
- 品質
- 保守性
- 再利用性
- トークン効率
- AIとの共同開発体験

を総合的に最大化することである。

---

# 現在AIを使っていること

## ① コードを書く

現在は

- 作りたいものを相談する
- 実装方針を考える
- コード生成
- 修正
- リファクタリング
- コードレビュー

などをAIと共同で行っている。

### 現状の課題

#### 課題1：長い会話で迷子になる

工程を複数フェーズに分けると、

- 特定工程だけ議論が長く続く
- 元の目的から逸脱する
- 今何をやっているか分からなくなる

ことが多い。

---

#### 課題2：引継ぎコンテキスト作成

新しいチャットへ移る際、

前チャットの内容を要約して引き継いでいるが、

- 少し情報が欠ける
- 表現が変わる
- 要約が悪い

だけでAIの理解が変わり、

意図しない方向へ進むことがある。

---

#### 課題3：コード品質を評価できない

AIが生成したコードについて、

- ベストプラクティスなのか
- 設計として美しいのか
- 保守性が高いのか
- 将来的に問題がないのか

を自分で十分評価できない。

---

#### 課題4：リファクタリング

リファクタリングや改良を、

より効率良く、
かつ品質を落とさず行いたい。

---

## ② トークン効率と知識管理

AIへ渡すコンテキスト量が増え続けている。

そのため、

トークン効率と品質を両立したい。

### 課題1：新しい知識の追加

現在は

- Webページ
- ドキュメント
- API仕様
- ホームページ

などをそのまま貼り付け、

「要約してください」

と依頼している。

しかし、

- 既存知識との重複
- 保存場所
- 更新方法

などが最適なのか分からない。

---

### 課題2：知識の継続的整理

現在は

「重複しているか確認してください」

「整合性を確認してください」

などを毎回直接依頼している。

しかし、

- 重複削除
- 古い情報の更新
- 矛盾の発見
- 全体構造の改善

についてもっと効率的な方法がある気がしている。

---

# 提案してほしいこと

私は以下に限定せず、

AI開発全体を改善できる提案を幅広く求めている。

---

# 1. Skills設計

現在考えているのは、

Claude Codeなどで利用する
再利用可能なSkillsである。

重要なのは、

**できるだけ汎用性を高めること。**

例えば

「リファクタリング」

というSkillは、

プロジェクトごとに

- 設計思想
- 命名規則
- アーキテクチャ

が異なるため、

そのままでは汎用性が低い。

そのため、

より抽象化し、

責務分割・階層化したSkillsへ落とし込みたい。

現在考えている候補は以下。

---

## Codebase Analysis

- プロジェクト構造把握
- エントリーポイント特定
- データフロー整理
- 依存関係分析
- 変更箇所分析

---

## Implementation Planning

- 要件整理
- 実装ステップ分割
- 影響範囲分析
- テスト計画

---

## Documentation Update

- README更新
- CHANGELOG更新
- AGENT.md更新
- Context更新
- APIドキュメント更新

---

## Code Review

- 可読性
- 保守性
- 性能
- セキュリティ
- 設計レビュー

---

## Refactoring

例として

- Rename
- Extract
- Split
- Cleanup

などへ細分化できると考えている。

---

私はこれら以外にも、

より適切な粒度、

階層構造、

Skill設計があれば積極的に提案してほしい。

---

# 2. Workflow設計

Skillsだけでなく、

Skillsをどう組み合わせるかも設計したい。

例えば、

新機能追加なら

Codebase Analysis

↓

Implementation Planning

↓

Implementation

↓

Review

↓

Refactor

↓

Documentation Update

↓

Context Update

のようなWorkflowも提案してほしい。

Workflowもできるだけ汎用化してほしい。

---

# 3. Context/RAG設計

以下について改善案を提案してほしい。

- AGENT.md
- context/
- project/
- source/
- RAG
- Vector DB

など。

特に

- 何をAGENT.mdへ残すべきか
- 何をRAGへ入れるべきか
- コンテキスト圧縮
- 長期運用方法

について知りたい。

---

# 4. Knowledge Management

知識管理全体について、

ベストプラクティスを提案してほしい。

例えば

- 新規知識追加
- 要約
- 差分更新
- 重複削除
- 整合性維持
- バージョン管理
- 情報寿命
- 検索性向上

など。

---

# 5. AIへの指示方法

プロンプト設計だけでなく、

AIとの共同開発全体における

- 指示の仕方
- タスク分割
- 会話の進め方
- コンテキストの渡し方
- レビュー依頼方法

などの改善案も提案してほしい。

---

# 6. アーキテクチャ提案

必要なら

- MCP
- RAG
- Knowledge Graph
- Workflow Engine
- Agent
- Subagent
- Memory
- ベクトルDB

なども含めて、

全体アーキテクチャを提案してほしい。

技術選定も歓迎する。

---

# 提案時に重視してほしいこと

提案する際は、

単なる思いつきではなく、

以下を意識してほしい。

- 汎用性
- 再利用性
- 保守性
- スケーラビリティ
- 長期運用
- AIとの相性
- トークン効率
- コンテキスト効率
- 学習コスト
- 導入コスト

について比較・評価してほしい。

また、

「SkillではなくWorkflowで解決すべき」

「これはContext設計の問題」

「これはRAGで解決すべき」

など、

責務分離の観点からも提案してほしい。

# 回答方針

私の案を前提とせず、

必要であれば根本から設計を見直して構わない。

「その設計自体が適切ではない」

という結論でも歓迎する。

部分最適ではなく、

AI開発環境全体を俯瞰した上で、

責務分離・再利用性・長期運用を重視した設計を提案してほしい。
