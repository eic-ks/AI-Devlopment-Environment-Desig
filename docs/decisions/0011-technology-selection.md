# ADR-0011: 統合アーキテクチャの技術要素採否とSubagentの位置づけ

- ステータス: 決定
- 日付: 2026-07-06
- 関連フェーズ: Phase 6（`requests/006-integrated-architecture.md`）

## コンテキスト

Phase 1〜5 で Skills・Workflow・Context/RAG・KB・指示方法を個別に設計した。Phase 6 はこれらを1つのアーキテクチャに統合するにあたり、CLAUDE.md が出発点として与えた7技術要素（MCP・RAG・Knowledge Graph・Workflow Engine・Agent/Subagent・Memory・ベクトルDB）の採否を確定する必要がある。放置すると、統合図に「いつか使うかもしれない箱」が残り続け、`docs/principles.md` 導入コスト軸が悪い例とする「最初から重い基盤をフル構築する提案」に滑り込む。

あわせて、Phase 5 がスコープ外として残した「Workflowの実行主体とAgent機能の対応づけ」のうち、統合図の作成に必要な最小限（Subagentの位置づけ）をここで決める。

## 検討した選択肢

判定基準は「この体系・このスケール（個人利用・ADR-0004）で要るか」だけであり、技術の一般的優劣は論じない。

| 選択肢 | 汎用性 | 保守性 | トークン効率 | 導入コスト | 備考 |
|---|---|---|---|---|---|
| A: ファイル基盤＋Subagentのみ採用（他は不採用・条件付き再検討） | 高（標準ツールのみ） | 高（写し・同期義務ゼロ） | 高（常駐増なし） | 低（今日から） | 採用案 |
| B: KG・ベクトルDB等を「将来枠」として部分導入 | 中 | 低（ファイルとの二重管理） | 中 | 高 | 個人スケールで監視値未達のまま保守負担だけ先行 |
| C: Workflow Engine導入で自動実行 | 低（特定基盤依存） | 中 | — | 高 | ゲート＝人裁定の設計と正面衝突 |

## 決定

**「個人スケールでファイルシステムを超える基盤は要らない」を既定とし、7要素を次のとおり仕分ける**（詳細は `outputs/architecture/02-technology-decisions.md`）。

| 要素 | 採否 | 一言理由 |
|---|---|---|
| Agent / Subagent | **採用** | ゲート区間内の作業実行者。下記の委譲禁止3点付き |
| Memory | 採用（実現済み） | L2状態層＋L4参照層が担う。ツール組込みの自動メモリは体系の正とせず補助 |
| RAG | 概念採用（実装はファイルベース） | L4＋検索手順（分類→glob→grep）が retrieval を担う。embedding型は不採用 |
| ベクトルDB | 不採用 | ADR-0002 の再確認。再検討条件（検索失敗件数・KBファイル数の監視）も同ADRのまま |
| Knowledge Graph | 不採用 | Phase 4 KB の代替にならず（関係の索引は書き込み規律・鮮度管理を代替しない）、補完としても関係情報の写し＝同期義務になる。再検討時も定型ヘッダの関係メタデータ拡充（軽量案）が先 |
| Workflow Engine | 不採用 | 中心は人のゲート裁定であり決定論的自動実行が目的でない。AIエージェント自身がL5定義のインタープリタ |
| MCP | 不採用（将来検討） | 体系はファイル＋標準ツールで完結。外部システムをL4の情報源に組み込む需要が出た時に再検討。**体系はMCPを禁じないが、体系の成立はMCPに依存しない** |

**Subagentの位置づけ**: メインセッション＝ゲート運営・指示出し・成果物レビュー・L2書き込み。Subagent＝ゲート区間内のステップ実行（成果物はファイルで返す）。**委譲禁止3点**——①ゲート裁定（人の判断点）②L1/L3/L5への書き込み（人間の手続き限定。Phase 3 の書き込み主体規定のまま）③F1/F2によるL2更新（現在地の正は1系統）。Subagentへの指示にも Phase 5 の3要素定型をそのまま使い、人→AIとAI→Subagentで規定を分けない。

## 結果・トレードオフ

- **不採用4要素はすべて条件付き**であり凍結ではない。再検討トリガー: ベクトルDB＝ADR-0002の監視値／KG＝関係起因の検索失敗の蓄積／MCP＝外部システム接続の需要発生／Workflow Engine＝明示的な再検討条件なし（ゲート規律の放棄を伴うため、変えるなら原則W1〜W4ごとADRで覆すこと）
- 自動メモリ機能を持つツール上では、体系の書き込み経路（Skill成果物）と自動メモリの二重記録が起きうる。体系側の正はあくまでL2/L4のファイルであり、矛盾したら自動メモリ側を疑う——この運用規律の形骸化検出は Phase 7 に委ねる
- Subagent委譲禁止3点は規約であり物理的強制ではない（W3停止規約と同じ限界）。破られた場合の検出も Phase 7 の棚卸しに載せる
- 検討過程・整合性照合の全体は `outputs/architecture/00-design-plan.md`（G1承認 2026-07-06）・`02-technology-decisions.md`・`03-consistency-check.md` を正とする
