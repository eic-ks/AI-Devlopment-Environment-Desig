# ADR-0005: 特定プロジェクトに寄せず抽象的な環境設計として進める

## ステータス

承認済み（Phase 0）

## コンテキスト

設計を進めるにあたり、特定のパイロットプロジェクトをベースにするか、抽象的に進めるかを決める必要があった。

## 決定

**特定プロジェクトに寄せず、抽象的な環境設計として進める。** 検証パイロットは未特定とする。

## 理由

- 特定プロジェクトに寄せると汎用性が下がる
- 「具体的なプロジェクトで検証」は設計が固まった後のフェーズで行うほうが効率的
- 抽象的な設計であっても、具体例・ユースケースは適宜示す

## 影響

- 設計は特定言語・フレームワーク・プロジェクト構造に依存しない
- 具体例はあくまで説明用であり、設計の前提にはしない
- 設計完了後に実プロジェクトでの検証フェーズを設ける余地を残す
