# ADR-0006: 実行可能Skillの構築（ADR-0003の部分改定）

- ステータス: 決定
- 日付: 2026-07-04
- 関連フェーズ: Phase 1（`requests/001-skills.md`）の追補
- 関連ADR: ADR-0003（成果物形式）を部分的に置き換え

## コンテキスト

ADR-0003では「成果物は設計ドキュメントのみ、実行可能SKILL.md雛形は作らない」としていた。Phase 1完了後、ユーザーから「実際のSkill構築までして欲しい」との要望があった。設計（6カテゴリ19Skill＋6原則）が確定済みのため、設計から実装への変換は機械的に行える状態にある。

## 検討した選択肢

| 選択肢 | 再利用性 | 導入コスト | 保守性 | 備考 |
|---|---|---|---|---|
| A: 設計ドキュメントのみ維持（現状） | — | 低 | 高 | 使うには毎回手動で読み込む必要があり、実利がない |
| B: ツール非依存の共通形式で実装 | 高 | 中 | 中 | どのツールでも動かず、結局変換作業が残る |
| C: Claude Code形式（SKILL.md）で実装（採用） | 中〜高 | 低 | 中 | 今の環境で即動く。Markdownベースのため他ツールへの変換も容易 |

## 決定

**Claude Code形式のSKILL.mdとして全19Skillを実装する（選択肢C）。**

- 置き場所: このリポジトリの `skills/<skill-name>/SKILL.md`（git管理）
- 利用経路: `~/.claude/skills/` へ各Skillをsymlinkし、全プロジェクトから利用可能にする
- 設計ドキュメント（`outputs/skills/`）が正、SKILL.mdは実装。設計を変えたらSKILL.mdに反映する（逆方向の変更は設計へ昇格させてから）
- 命名は設計のSkill名を踏襲。ただしClaude Code組み込みスキルとの衝突を避けるため2つだけ改名: D1 code-review → `checklist-code-review`、C4 cleanup → `code-cleanup`
- 設計上「Skill定義単体では担保できない」とした「前提入力の欠落時に進めずに要求する」挙動は、各SKILL.md本文に停止指示として明記することで実装レベルで担保する（ADR-0001の持ち越し事項への部分回答）

## 結果・トレードオフ

- **ツール非依存性を一部手放した**。Claude Codeの仕様変更に追従するコストが発生する。緩和策: ツール非依存の設計ドキュメントを正として維持し、SKILL.mdは薄い変換層に留める
- **二重管理のリスク**。設計（outputs/skills/）と実装（skills/）の同期ずれが起こりうる。緩和策: 変更は必ず設計→実装の一方向とし、ADR-0001の分割基準を変える場合はADRを先に更新する
- パイロット未特定の前提（ADR-0005）は維持する。SKILL.mdは特定プロジェクトに依存しない形で書く（注入原則の実装）
- Phase 2以降の成果物形式は引き続き設計ドキュメントを基本とし、実装するかは都度判断する（本ADRはPhase 1のSkillsに限る）
