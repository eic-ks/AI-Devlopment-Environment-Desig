# ADR-0004: 想定利用シーンは個人（ソロ開発者）とする

## ステータス

承認済み（Phase 0）

## コンテキスト

設計の想定利用者を決める必要があった。選択肢として以下があった：

1. 個人（ソロ開発者）
2. 小規模チーム
3. 組織全体

## 決定

**個人（ソロ開発者としての自分専用）を想定する。** チーム利用は将来の拡張として考慮するが、主要な設計判断は個人利用に最適化する。

## 理由

- 現在の課題はすべて個人の開発体験に関するもの
- チーム利用を想定すると、権限管理・共有・同期といった別次元の複雑さが入り込む
- 個人で十分に練られた設計は、後からチーム向けに拡張しやすい

## 影響

- 権限管理・マルチユーザー対応は設計スコープ外
- 個人のワークフローに最適化した設計判断を優先する
- スケーラビリティ評価は「知識量・Skills数の増加」に焦点を当てる
