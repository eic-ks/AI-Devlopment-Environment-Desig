# ADR-0003: 成果物の形式は設計ドキュメントのみとする

## ステータス

承認済み（Phase 0）→ **部分改定あり**（ADR-0006・ADR-0009 を参照）

> 改定後の現行方針: Phase 1 Skills（`skills/`）・Phase 2 Workflows（`workflows/`）・Phase 3/4 scaffold（`scaffold/`）は Claude Code 形式で実装済み。判断基準・原則系の文書は本ADRのとおり設計ドキュメントのまま。以下の本文は Phase 0 時点の決定の記録である。

## コンテキスト

AI開発環境の全体設計を進めるにあたり、成果物の形式を決める必要があった。選択肢として以下があった：

1. 設計ドキュメントのみ
2. 実行可能な SKILL.md 雛形やコードも含める
3. プロトタイプ実装まで行う

## 決定

**設計ドキュメントのみを成果物とする。** 実行可能な SKILL.md 雛形やコードは作成しない。

## 理由

- 設計が固まっていない段階で実装に手を出すと、設計を実装に合わせてしまうリスクがある
- まず「何をどう設計すべきか」を十分に議論し、その後に実装フェーズを別途設けるほうが品質が高い
- 設計ドキュメントであれば、ツール非依存で議論・レビューできる

## 影響

- すべてのフェーズで成果物は `.md` ファイルの設計ドキュメントとなる
- 実装・コード生成は本プロジェクトのスコープ外
- 設計完了後に別途実装フェーズを設ける想定
