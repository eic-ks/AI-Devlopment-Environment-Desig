# research/ — リサーチ・一次情報の管理

## ディレクトリ構成

| ディレクトリ | 用途 | 例 |
|---|---|---|
| `articles/` | 外部記事・ブログポスト・論文の要約 | ベストプラクティス記事、技術ブログ |
| `notes/` | 自分の考察メモ・実験記録 | 設計の比較検討メモ、試行結果 |
| `references/` | 公式ドキュメント・API仕様の要約 | Claude Code ドキュメント、MCP仕様 |

## ファイル命名規則

```
{日付}-{短い説明}.md

例:
2026-07-03-claude-code-skills-best-practices.md
2026-07-03-rag-architecture-comparison.md
```

## 新規追加時の手順

1. 対応するサブディレクトリにファイルを作成する
2. **重複確認**: 追加前に同ディレクトリ内の既存ファイルを確認し、同じ情報源・同じトピックがないか確認する
3. ファイル冒頭に以下のメタ情報を記載する：

```markdown
# {タイトル}

- 情報源: {URL or 出典}
- 取得日: {YYYY-MM-DD}
- 関連フェーズ: Phase {N}
- 要約者: {AI / 人間}

## 要約

（ここに要約を記載）

## 要点

- ...
```

4. 既存ファイルと内容が重複する場合は、既存ファイルに追記・統合し、新規ファイルは作成しない

## 情報の鮮度管理

- `取得日` が6ヶ月以上前の情報は、利用前に最新版を確認すること
- 古くなった情報は削除せず、ファイル冒頭に `> [!WARNING] この情報は {日付} 時点のものです。最新版を確認してください。` を追記する
