## requests/005-instruction-methods.md「検討すべき問い」との対応

| 問い | 答えの所在 |
|---|---|
| タスクをAIに渡す時、どの粒度まで分割してから渡すべきか | `02-task-splitting-criteria.md` 1〜3章。人は事前分割しない。シナリオ判定＋渡す単位（上限1ゲート区間・下限1Skill）のみ |
| 会話が長くなりそうな時、事前にどう区切っておくべきか | `02-task-splitting-criteria.md` 4章。事前設計不要・ゲート駆動。一般化＝「後続の前提が固定される点で切る」 |
| コンテキストを渡す時、「貼り付け」「ファイル参照」「要約」のどれを使い分けるか | `03-context-passing.md` 全体。ファイル参照が既定・貼り付けは条件付き例外・要約は伝達場面でも禁止 |
| コードレビューをAIに依頼する時、どういう依頼の仕方だと精度・再現性が上がるか | `04-review-request-template.md` 全体。精度は観点ファイルの質が決める・再現性は定型化が決める。育てるループで改善 |
| `docs/handoff.md` のような構造化引継ぎは日常のコーディングタスクにも適用できるか | `02-task-splitting-criteria.md` 5章。できる。発動条件はゲート区間2つ以上にまたがる見込み（シナリオ判定の副産物として決まる） |
