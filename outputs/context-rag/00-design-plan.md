# Phase 3（Context/RAG設計）設計方針・骨子

作成: 2026-07-05 メインセッション。`requests/003-rag.md` の問い5つへの回答方針と成果物骨子。
G1（計画承認）: 2026-07-05 ユーザー承認済み（再検討の指示を受け、下記判断5・6を精緻化した上で確定）。

ユーザー確認済みの前提: ①日常のCLAUDE.mdは**肥大化している**（規約・手順・経緯が混在） ②RAG基盤は**未導入・まっさら**。

## 中心となる設計判断

### 判断1: 5層の配置モデル

`docs/principles.md` の責務分離フレームを「いつ読み込まれるか」で具体化する。

| 層 | 置くもの | 読み込みタイミング |
|---|---|---|
| L1 常駐層（AGENT.md） | 行動規約（停止規約W3等）＋状態ファイルへの入口＋各層の場所と引き方（ポインタのみ） | 毎セッション自動 |
| L2 状態層（current-state / handoff） | 現在地・引継ぎ | セッション開始時（F3経由） |
| L3 注入層（context/） | プロジェクト固有の判断基準：命名規則・観点チェックリスト・テンプレート・テストコマンド | Workflowステップ開始時、Skill前提入力【必須】に応じ該当ファイルのみ |
| L4 参照層（knowledge/） | 事実・経緯・蓄積知見：過去のバグ対処・ハマりどころ・ADR詳細 | 必要時に検索（E3が書き込む） |
| L5 定義層（skills/ workflows/） | Skill・Workflow定義 | 起動時のみ |

- L3とL4の区別 = 「現在有効な判断基準（規約）」vs「蓄積される事実・経緯」。Phase 1責務境界の判定例3・4（→L3）と5（→L4）に整合
- Phase 1/2持ち越しの「固有知識の恒久的な置き場」への回答 = **L3 注入層**

### 判断2: AGENT.mdに残す基準 = 3テスト＋分量予算

次のいずれかを満たすものだけがL1に残れる。1つも満たさなければ追い出す。

- **T1 常時性**: どのタスクのセッションでも参照するか
- **T2 誤動作防止**: 読み込まれないとAIの挙動が壊れる規約か（停止規約・推測補完禁止等）
- **T3 地図性**: 内容そのものではなく、置き場所へのポインタか

＋**分量予算**（目安値を成果物で規定。超過が棚卸しのトリガー）。肥大化解消は「削除」ではなく「L3/L4へ移してポインタ化」で行う。

### 判断3: RAG = ファイルベースKB＋エージェント検索。Vector DBは不採用（ADR-0002）

- 理由: ①L3/L4は既知の構造（カテゴリ・名前）で引けるためセマンティック検索の主要ユースケースが無い ②ソロ開発者の知識量ならgrep/globで十分 ③embedding更新・基盤保守のコストが便益を上回る（評価軸: 導入コスト） ④エージェントはツールで反復検索できるためone-shot検索基盤が不要
- 再検討条件を明記: 「名前で引けない知見の検索」が繰り返し失敗する／KBが数千ファイル規模になる、等

### 判断4: 圧縮は「構造化」のみ。「要約」による圧縮を禁止

原点課題2（要約の劣化）への一貫回答。F系の定型テンプレート充填＋ファイル外部化＋ポインタ化だけを圧縮手段とする。Phase 1原則2・原則5例外（F系厳格）の延長。

### 判断5: W3停止規約の実行時強制 = L1の規約＋L3のマニフェスト

- L1（常駐層）に停止規約を置く（毎セッション必ず読み込まれる＝強制の足場）
- L3に**知識マニフェスト**（`context/INDEX.md`）を置く。Skill前提入力【必須】→ファイルパスの対応表の**唯一の置き場**とする（L1は「マニフェストの場所」を指すだけ。地図の実体を2箇所に持たない＝保守性「直す場所は1箇所」）
- ステップ開始条件の充足確認を「マニフェストの該当パスにファイルが存在し空でないか」という機械的判定に還元（原則W5「判定はファイルの存在」と同型）。欠落→停止して要求

### 判断6: メンテナンスサイクル

- **書き込み規律**: 新知識の既定の行き先はL4（AGENT.md直行禁止）。層を上げるには下記の昇格条件を要する
- **昇格**: ①ファイル外から都度手渡しされた知識（会話で毎回説明している規約等）が繰り返されたら→L3へ収載しマニフェストに登録 ②L3の知識がタスクを問わず毎セッション必要になったら→3テスト合格を条件にL1へ昇格
- **降格**: 参照されないL1項目はL3/L4へ、規約として失効したL3項目は経緯としてL4へ（または削除）
- **棚卸しトリガー**: L1分量予算の超過時＋フェーズ/マイルストーン完了時（G3に紐づけ）
- **鮮度**: 各知識ファイルに更新日メタデータ。実態との矛盾を発見したら即時修正または削除（放置禁止）
